# 01 — 开发环境搭建

## 1.1 宿主机环境

推荐使用 Ubuntu 18.04/20.04 作为开发宿主机（可以用虚拟机或 WSL2）。

```bash
# 基础开发工具
sudo apt update
sudo apt install -y build-essential cmake git wget unzip \
    autoconf automake libtool pkg-config \
    python3 python3-pip \
    libncurses5-dev u-boot-tools device-tree-compiler \
    gdb-multiarch
```

---

## 1.2 交叉编译工具链

### 方式一：使用野火官方 SDK（推荐）

野火 STM32MP157 提供了完整的 Yocto SDK，包含工具链和 sysroot。

```bash
# 1. 从野火资料包中找到 SDK 安装脚本（通常在开发板配套资料中）
# 文件名类似：st-image-weston-openstlinux-weston-stm32mp1-x86_64-toolchain-3.1-xxx.sh
chmod +x st-image-weston-*.sh
./st-image-weston-*.sh
# 默认安装到 /opt/st/stm32mp1/

# 2. 每次开发前 source 环境变量
source /opt/st/stm32mp1/environment-setup-cortexa7t2hf-neon-vfpv4-ostl-linux-gnueabi
# 验证
echo $CC
# 应输出类似：arm-ostl-linux-gnueabi-gcc ...
$CC --version
```

### 方式二：使用 Linaro 工具链

如果没有野火 SDK，可以用通用 ARM 工具链：

```bash
# 下载 Linaro 工具链
wget https://releases.linaro.org/components/toolchain/binaries/7.5-2019.12/arm-linux-gnueabihf/gcc-linaro-7.5.0-2019.12-x86_64_arm-linux-gnueabihf.tar.xz
tar xf gcc-linaro-7.5.0-2019.12-x86_64_arm-linux-gnueabihf.tar.xz -C /opt/

# 添加到 PATH
export CROSS_COMPILE=/opt/gcc-linaro-7.5.0-2019.12-x86_64_arm-linux-gnueabihf/bin/arm-linux-gnueabihf-
export CC=${CROSS_COMPILE}gcc
export CXX=${CROSS_COMPILE}g++
```

## 1.3 交叉编译 OpenCV

开发板上需要 OpenCV，必须交叉编译 ARM 版本。

```bash
# 1. 下载 OpenCV 源码
cd ~/workspace
git clone https://github.com/opencv/opencv.git -b 4.5.5 --depth 1
mkdir opencv/build_arm && cd opencv/build_arm

# 2. 交叉编译（确保已 source SDK 环境变量）
cmake .. \
    -DCMAKE_BUILD_TYPE=Release \
    -DCMAKE_INSTALL_PREFIX=/opt/opencv_arm \
    -DCMAKE_TOOLCHAIN_FILE=../platforms/linux/arm-gnueabi.toolchain.cmake \
    -DENABLE_NEON=ON \
    -DENABLE_VFPV3=ON \
    -DBUILD_TESTS=OFF \
    -DBUILD_PERF_TESTS=OFF \
    -DBUILD_EXAMPLES=OFF \
    -DBUILD_opencv_python2=OFF \
    -DBUILD_opencv_python3=OFF \
    -DWITH_GTK=OFF \
    -DWITH_QT=OFF \
    -DWITH_V4L=ON \
    -DWITH_LIBV4L=ON \
    -DWITH_FFMPEG=ON \
    -DCMAKE_C_COMPILER=$CC \
    -DCMAKE_CXX_COMPILER=$CXX

# 3. 编译安装
make -j$(nproc)
make install

# 4. 验证
ls /opt/opencv_arm/lib/
# 应该看到 libopencv_core.so, libopencv_imgproc.so 等
```

### 如果使用野火 SDK

野火 SDK 的 sysroot 中可能已经包含 OpenCV，检查一下：

```bash
ls $OECORE_TARGET_SYSROOT/usr/lib/libopencv*
# 如果存在，就不需要自己编译
```

## 1.4 交叉编译 Qt5

### 方式一：使用 SDK 自带的 Qt（推荐）

野火 STM32MP157 的 Yocto SDK 通常已包含 Qt5：

```bash
source /opt/st/stm32mp1/environment-setup-cortexa7t2hf-neon-vfpv4-ostl-linux-gnueabi

# 检查 qmake 是否可用
which qmake
qmake --version
# 应输出 Qt 5.x.x
```

### 方式二：手动交叉编译 Qt5

如果 SDK 不含 Qt，需要手动编译：

```bash
# 1. 下载 Qt 源码
cd ~/workspace
wget https://download.qt.io/archive/qt/5.12/5.12.12/single/qt-everywhere-src-5.12.12.tar.xz
tar xf qt-everywhere-src-5.12.12.tar.xz
cd qt-everywhere-src-5.12.12

# 2. 创建交叉编译配置文件
# 复制一份 mkspec
cp -r qtbase/mkspecs/linux-arm-gnueabi-g++ qtbase/mkspecs/linux-stm32mp157-g++

# 3. 编辑 qtbase/mkspecs/linux-stm32mp157-g++/qmake.conf
cat > qtbase/mkspecs/linux-stm32mp157-g++/qmake.conf << 'EOF'
include(../common/linux.conf)
include(../common/gcc-base-unix.conf)
include(../common/g++-unix.conf)

QMAKE_CC                = arm-linux-gnueabihf-gcc
QMAKE_CXX               = arm-linux-gnueabihf-g++
QMAKE_LINK              = arm-linux-gnueabihf-g++
QMAKE_LINK_SHLIB        = arm-linux-gnueabihf-g++
QMAKE_AR                = arm-linux-gnueabihf-ar cqs
QMAKE_OBJCOPY           = arm-linux-gnueabihf-objcopy
QMAKE_NM                = arm-linux-gnueabihf-nm -P
QMAKE_STRIP             = arm-linux-gnueabihf-strip
load(qt_config)
EOF

# 4. 配置
./configure \
    -prefix /opt/qt5_arm \
    -xplatform linux-stm32mp157-g++ \
    -opensource -confirm-license \
    -release \
    -no-opengl \
    -linuxfb \
    -nomake examples \
    -nomake tests \
    -no-dbus

# 5. 编译安装
make -j$(nproc)
make install
```

## 1.5 宿主机 Qt Creator 配置（可选，用于开发调试）

如果你想在 Qt Creator 中开发，然后交叉编译部署到开发板：

1. 打开 Qt Creator → 工具 → 选项 → Kits
2. **编译器**：添加交叉编译器路径（arm-linux-gnueabihf-g++）
3. **Qt Versions**：添加交叉编译的 qmake 路径（/opt/qt5_arm/bin/qmake）
4. **设备**：添加 Generic Linux Device，填入开发板 IP
5. **Kit**：新建一个 Kit，关联上面的编译器、Qt 版本、设备

## 1.6 文件传输方式

开发过程中需要频繁把编译好的程序传到开发板：

```bash
# 方式一：scp（推荐）
scp endoscope_demo root@192.168.1.xxx:/opt/app/

# 方式二：NFS 共享目录（开发效率最高）
# 宿主机
sudo apt install nfs-kernel-server
echo "/home/你的用户名/nfs_share *(rw,sync,no_root_squash)" | sudo tee -a /etc/exports
sudo mkdir -p /home/你的用户名/nfs_share
sudo exportfs -arv

# 开发板
mount -t nfs 192.168.1.xxx:/home/你的用户名/nfs_share /mnt/nfs -o nolock

# 方式三：SD 卡拷贝（最简单但效率低）
```

## 1.7 环境验证清单

在进入下一步之前，确认以下全部通过：

- [ ] 交叉编译工具链可用：`$CC --version` 输出 ARM gcc 版本
- [ ] 交叉编译 OpenCV 完成：`/opt/opencv_arm/lib/` 下有 .so 文件
- [ ] 交叉编译 Qt5 完成：`/opt/qt5_arm/bin/qmake --version` 正常输出
- [ ] 能写一个 Hello World 交叉编译并在开发板上运行
- [ ] 宿主机与开发板网络互通（ping 通）
- [ ] 文件传输方式确认可用（scp 或 NFS）
