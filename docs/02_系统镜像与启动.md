# 02 — 系统镜像与启动

## 2.1 烧录系统镜像

### 使用野火官方镜像

野火 STM32MP157 提供了预编译的 Linux 镜像（基于 OpenSTLinux / Buildroot）。

```bash
# 1. 从野火资料包获取镜像文件
# 通常包含：
#   - tf-a (Trusted Firmware)
#   - u-boot
#   - kernel (zImage + dtb)
#   - rootfs (ext4)

# 2. 烧录方式一：STM32CubeProgrammer（推荐）
# 下载安装：https://www.st.com/en/development-tools/stm32cubeprog.html
# 开发板拨码开关设为 USB 烧录模式（参考野火文档）
# 用 USB 线连接开发板 USB OTG 口到电脑
# 打开 STM32CubeProgrammer，选择 USB 连接，加载 tsv 烧录配置文件

# 3. 烧录方式二：SD 卡启动
# 使用 dd 命令或 balenaEtcher 将镜像写入 SD 卡
sudo dd if=sdcard.img of=/dev/sdX bs=8M conv=fsync status=progress
```

### 确认系统启动

```bash
# 串口连接开发板（115200 波特率）
sudo minicom -D /dev/ttyUSB0 -b 115200

# 或使用 screen
sudo screen /dev/ttyUSB0 115200

# 开发板上电后应看到 U-Boot 启动日志，然后进入 Linux shell
# 登录：root（通常无密码或密码为 root）
```

## 2.2 网络配置

```bash
# 开发板上配置有线网络
# 方式一：DHCP 自动获取
udhcpc -i eth0
# 或
dhclient eth0

# 方式二：静态 IP
ifconfig eth0 192.168.1.100 netmask 255.255.255.0 up
route add default gw 192.168.1.1

# 验证
ping 192.168.1.xxx  # ping 宿主机
```

## 2.3 确认 OV5640 驱动

这是最关键的一步。STM32MP157 通过 DCMI（Digital Camera Memory Interface）接口连接 OV5640。

### 检查设备树

OV5640 需要在设备树中正确配置。野火开发板通常已经配好了，但需要确认。

```bash
# 在开发板上检查设备树
ls /proc/device-tree/soc/
# 应该能看到 dcmi 相关节点

# 查看 DCMI 节点
cat /proc/device-tree/soc/dcmi@4c006000/status
# 应输出 "okay"

# 查看 I2C 总线上的 OV5640
ls /proc/device-tree/soc/i2c@*/ov5640*/
# 或
cat /proc/device-tree/soc/i2c@*/ov5640*/status
```

### 如果设备树未配置 OV5640

需要修改设备树源文件（.dts）。以下是典型的 OV5640 设备树配置：

```dts
/* 在对应的 i2c 节点下添加 */
&i2c1 {
    status = "okay";
    clock-frequency = <100000>;

    ov5640: camera@3c {
        compatible = "ovti,ov5640";
        reg = <0x3c>;
        clocks = <&clk_ext_camera>;
        clock-names = "xclk";
        DOVDD-supply = <&v2v8>;
        powerdown-gpios = <&gpioa 4 GPIO_ACTIVE_HIGH>;
        reset-gpios = <&gpioa 3 GPIO_ACTIVE_LOW>;

        port {
            ov5640_ep: endpoint {
                remote-endpoint = <&dcmi_ep>;
                bus-width = <8>;
                data-shift = <2>;
                hsync-active = <0>;
                vsync-active = <0>;
                pclk-sample = <1>;
            };
        };
    };
};

/* DCMI 节点 */
&dcmi {
    status = "okay";
    pinctrl-names = "default", "sleep";
    pinctrl-0 = <&dcmi_pins_a>;
    pinctrl-1 = <&dcmi_sleep_pins_a>;

    port {
        dcmi_ep: endpoint {
            remote-endpoint = <&ov5640_ep>;
            bus-width = <8>;
            pclk-sample = <1>;
        };
    };
};
```

**注意**：具体的 GPIO 引脚、I2C 总线编号、电源域要参考野火开发板原理图。

### 编译设备树

```bash
# 在宿主机上（需要内核源码）
# 方式一：单独编译 dtb
dtc -I dts -O dtb -o stm32mp157-fire.dtb stm32mp157-fire.dts

# 方式二：在内核源码中编译
cd linux-source/
make ARCH=arm CROSS_COMPILE=arm-linux-gnueabihf- dtbs

# 将编译好的 dtb 拷贝到开发板 /boot/ 目录
scp stm32mp157-fire.dtb root@192.168.1.100:/boot/
# 重启开发板
```

## 2.4 验证摄像头

### 检查驱动加载

```bash
# 检查内核模块
lsmod | grep ov5640
# 应看到 ov5640 模块

# 如果没有自动加载
modprobe ov5640
modprobe stm32_dcmi

# 检查 video 设备
ls /dev/video*
# 应看到 /dev/video0

# 查看内核日志
dmesg | grep -i "ov5640\|dcmi\|video"
# 应看到类似：
# ov5640 1-003c: ov5640_read_reg: error reading reg 0x300a
# 或成功信息：
# ov5640 1-003c: ov5640 Product ID 56 40
```

### 使用 v4l2-ctl 测试

```bash
# 安装 v4l2-utils（如果系统中没有）
# 如果是 Buildroot 系统，可能需要在 menuconfig 中开启
# 如果是 Debian/Ubuntu 系统：
apt install v4l-utils

# 查看摄像头信息
v4l2-ctl --device=/dev/video0 --all

# 列出支持的像素格式
v4l2-ctl --device=/dev/video0 --list-formats-ext

# 期望看到类似输出：
# [0]: 'YUYV' (YUYV 4:2:2)
#     Size: Discrete 640x480
#     Size: Discrete 1280x720
#     Size: Discrete 1920x1080

# 抓一帧测试
v4l2-ctl --device=/dev/video0 \
    --set-fmt-video=width=640,height=480,pixelformat=YUYV \
    --stream-mmap=4 \
    --stream-count=1 \
    --stream-to=test_frame.raw

# 验证文件大小
ls -la test_frame.raw
# 640*480*2 = 614400 字节（YUYV 每像素2字节）
```

### 使用 GStreamer 测试实时预览（可选）

```bash
# 如果系统中有 GStreamer
gst-launch-1.0 v4l2src device=/dev/video0 ! \
    video/x-raw,width=640,height=480,framerate=30/1 ! \
    videoconvert ! \
    fbdevsink
# 如果屏幕上能看到摄像头画面，说明整个链路通了
```

## 2.5 确认 LCD 显示

```bash
# 检查 framebuffer 设备
ls /dev/fb*
# 应看到 /dev/fb0

# 查看 framebuffer 信息
cat /sys/class/graphics/fb0/virtual_size
# 输出类似：800,480

cat /sys/class/graphics/fb0/bits_per_pixel
# 输出：16 或 32

# 简单测试：填充红色
dd if=/dev/urandom of=/dev/fb0 bs=800 count=480

# 如果屏幕显示花屏（随机色块），说明 framebuffer 工作正常
```

### 确认 Qt 能在 LCD 上显示

```bash
# 如果使用 linuxfb 插件
export QT_QPA_PLATFORM=linuxfb:fb=/dev/fb0
# 或 eglfs（如果有 GPU 支持）
export QT_QPA_PLATFORM=eglfs

# 运行一个 Qt 自带的示例程序测试
/opt/qt5_arm/examples/widgets/widgets/analogclock/analogclock
# 屏幕上应该显示一个时钟
```

## 2.6 本步骤验证清单

- [ ] 系统镜像烧录成功，开发板正常启动进入 Linux shell
- [ ] 网络配置完成，宿主机与开发板互通
- [ ] `/dev/video0` 存在，`dmesg` 中有 OV5640 识别成功的日志
- [ ] `v4l2-ctl --all` 能正常输出摄像头信息
- [ ] `v4l2-ctl --stream-to` 能抓到一帧数据，文件大小正确
- [ ] `/dev/fb0` 存在，LCD 屏幕能显示内容
- [ ] Qt 程序能在 LCD 上正常显示
