# V4L2 学习文档（二）：v4l2-ctl 命令详解

> v4l2-ctl 是 V4L2 的命令行调试工具，相当于摄像头的"万用表"。
> 写代码之前先用它确认硬件能力，写代码之后用它对比验证。

---

## 1. 必须掌握的命令（日常开发用）

### 1.1 查看所有视频设备

```bash
v4l2-ctl --list-devices
```

输出示例：
```
STM32 Camera Memory Interface (platform:dcmi):
        /dev/video0
```

**用途**：确认摄像头设备节点是 `/dev/video0` 还是其他编号。
有时候系统中有多个视频设备（摄像头、HDMI 采集卡等），需要确认用哪个。

### 1.2 查看摄像头全部信息

```bash
v4l2-ctl --device=/dev/video0 --all
# 简写
v4l2-ctl -d /dev/video0 --all
```

**用途**：一次性看到驱动信息、当前格式、所有 Controls。开发初期最常用。

### 1.3 查看支持的格式和分辨率

```bash
v4l2-ctl -d /dev/video0 --list-formats-ext
```

**用途**：确定代码中应该用什么像素格式、分辨率、帧率。

### 1.4 设置采集格式

```bash
# 设置为 640x480 YUYV
v4l2-ctl -d /dev/video0 --set-fmt-video=width=640,height=480,pixelformat=YUYV
```

**用途**：在抓帧之前设置格式。也可以用来测试某个格式是否真的能用。

### 1.5 查看当前格式

```bash
v4l2-ctl -d /dev/video0 --get-fmt-video
```

**用途**：确认 set-fmt 是否生效。有时候硬件会把你请求的分辨率调整为最接近的支持值。

### 1.6 抓帧（最重要的调试命令）

```bash
# 抓 1 帧 YUYV 数据，保存到文件
v4l2-ctl -d /dev/video0 \
    --set-fmt-video=width=640,height=480,pixelformat=YUYV \
    --stream-mmap=4 \
    --stream-count=1 \
    --stream-to=/tmp/test_frame.raw
```

**参数解释：**

```
--set-fmt-video=width=640,height=480,pixelformat=YUYV
│               │          │              │
│               │          │              └─ 像素格式：YUYV
│               │          └─ 高度：480
│               └─ 宽度：640
└─ 设置视频采集格式

--stream-mmap=4
│         │    │
│         │    └─ 使用 4 个缓冲区
│         └─ 使用 mmap 方式（内存映射，效率最高）
└─ 开始流式采集

--stream-count=1
│              │
│              └─ 只采集 1 帧
└─ 采集帧数

--stream-to=/tmp/test_frame.raw
│           │
│           └─ 保存到这个文件
└─ 将采集的数据写入文件
```

**验证抓帧结果：**
```bash
ls -la /tmp/test_frame.raw
# YUYV 格式：文件大小 = width × height × 2
# 640 × 480 × 2 = 614400 字节

# JPEG 格式：文件大小不固定（压缩后），通常几十 KB
# RGB565 格式：文件大小 = width × height × 2 = 614400 字节
```

### 1.7 设置和查看 Controls

```bash
# 查看所有 controls
v4l2-ctl -d /dev/video0 --list-ctrls

# 查看详细信息（包括 flags）
v4l2-ctl -d /dev/video0 --list-ctrls-menus

# 设置对比度为 128
v4l2-ctl -d /dev/video0 --set-ctrl=contrast=128

# 水平翻转
v4l2-ctl -d /dev/video0 --set-ctrl=horizontal_flip=1

# 关闭自动曝光，然后手动设置曝光值
v4l2-ctl -d /dev/video0 --set-ctrl=auto_exposure=1
v4l2-ctl -d /dev/video0 --set-ctrl=exposure=500

# 查看当前某个 control 的值
v4l2-ctl -d /dev/video0 --get-ctrl=contrast
```

**用途**：在写代码之前，先用命令行测试 control 是否有效。
比如你想在代码中加"翻转"功能，先用命令行试一下 `--set-ctrl=horizontal_flip=1`，
看看画面是否真的翻转了，再写代码。

### 1.8 设置帧率

```bash
# 查看当前帧率
v4l2-ctl -d /dev/video0 --get-parm

# 设置帧率为 30fps
v4l2-ctl -d /dev/video0 --set-parm=30
```

## 2. 进阶命令（偶尔用到）

### 2.1 连续抓帧测试帧率

```bash
# 连续采集 100 帧，不保存，只看帧率
v4l2-ctl -d /dev/video0 \
    --set-fmt-video=width=640,height=480,pixelformat=YUYV \
    --stream-mmap=4 \
    --stream-count=100
```

输出会显示每帧的时间戳和帧率，用来验证实际帧率是否达标。

### 2.2 查看支持的 Controls 菜单项

```bash
# 比如 power_line_frequency 是 menu 类型
v4l2-ctl -d /dev/video0 --list-ctrls-menus
```

输出：
```
power_line_frequency 0x00980918 (menu) : min=0 max=3 default=1 value=1
    0: Disabled
    1: 50 Hz
    2: 60 Hz
    3: Auto
```

### 2.3 查看摄像头支持的所有分辨率（不含帧率）

```bash
v4l2-ctl -d /dev/video0 --list-framesizes=YUYV
```

## 3. 命令与代码的对应关系

这是最重要的部分——每个 v4l2-ctl 命令背后对应的 C 代码 ioctl 调用：

| v4l2-ctl 命令 | 对应的 ioctl | 代码中的函数 |
|---------------|-------------|-------------|
| `--all` (Driver Info) | `VIDIOC_QUERYCAP` | queryCapability() |
| `--list-formats-ext` | `VIDIOC_ENUM_FMT` + `VIDIOC_ENUM_FRAMESIZES` | enumFormats() |
| `--set-fmt-video` | `VIDIOC_S_FMT` | setFormat() |
| `--get-fmt-video` | `VIDIOC_G_FMT` | — |
| `--stream-mmap=4` | `VIDIOC_REQBUFS` + `mmap` | requestBuffers() |
| 开始采集 | `VIDIOC_STREAMON` | startStreaming() |
| 抓一帧 | `VIDIOC_DQBUF` | grabFrame() |
| 帧放回队列 | `VIDIOC_QBUF` | grabFrame() |
| 停止采集 | `VIDIOC_STREAMOFF` | stopStreaming() |
| `--set-ctrl=contrast=128` | `VIDIOC_S_CTRL` | setControl() |
| `--get-ctrl=contrast` | `VIDIOC_G_CTRL` | getControl() |
| `--set-parm=30` | `VIDIOC_S_PARM` | setFrameRate() |

**理解这个对应关系后**，你就知道：
- v4l2-ctl 能做的事，代码里都能做
- 先用 v4l2-ctl 验证，再写代码实现
- 代码出问题时，用 v4l2-ctl 对比排查

## 4. 调试技巧

### 摄像头不工作时的排查顺序

```
1. ls /dev/video*
   → 没有设备？检查驱动和设备树

2. v4l2-ctl -d /dev/video0 --all
   → 能输出信息？说明驱动正常
   → 报错？说明设备节点有问题

3. v4l2-ctl -d /dev/video0 --list-formats-ext
   → 能列出格式？说明摄像头通信正常
   → 空的？说明 I2C 通信失败，检查硬件连接

4. v4l2-ctl --stream-mmap=4 --stream-count=1 --stream-to=test.raw
   → 能抓到帧？说明采集链路通
   → 超时？说明 DCMI 时序有问题

5. 检查文件大小
   → 大小正确？硬件完全正常
   → 大小为 0？数据传输有问题
```
