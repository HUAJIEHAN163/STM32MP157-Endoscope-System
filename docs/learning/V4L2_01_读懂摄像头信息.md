# V4L2 学习文档（一）：如何读懂摄像头信息

> 本文档以我们实际的 OV5640 + STM32MP157 为例，逐行解读 v4l2-ctl 的输出，
> 教你如何从一堆信息中提取出写代码需要的关键参数。

---

## 1. 两个关键命令

在开发板上，获取摄像头信息只需要两个命令：

```bash
v4l2-ctl --device=/dev/video0 --all              # 查看摄像头全部信息
v4l2-ctl --device=/dev/video0 --list-formats-ext  # 查看支持的格式和分辨率
```

下面逐段解读输出。

---

## 2. 解读 `--all` 输出

### 2.1 驱动信息（Driver Info）

```
Driver Info:
        Driver name      : stm32-dcmi
        Card type        : STM32 Camera Memory Interface
        Bus info         : platform:dcmi
        Driver version   : 4.19.94
```

**你需要关注的：**
- `Driver name: stm32-dcmi` → 驱动是 STM32 的 DCMI（Digital Camera Memory Interface）
  - 这告诉你底层走的是 DCMI 接口，不是 USB UVC
  - 如果是 USB 摄像头，这里会显示 `uvcvideo`
- `Card type` → 设备描述，确认是摄像头而不是其他视频设备

**写代码时的影响：** 无直接影响，但调试时如果 Driver name 不对，说明加载了错误的驱动。

### 2.2 设备能力（Capabilities）

```
        Capabilities     : 0x85200001
                Video Capture
                Read/Write
                Streaming
                Extended Pix Format
                Device Capabilities
```

**你需要关注的：**

| 能力 | 含义 | 对代码的影响 |
|------|------|-------------|
| **Video Capture** | 支持视频采集 | 基本要求，没有这个就不是摄像头 |
| **Read/Write** | 支持 read() 系统调用读取帧 | 可以用最简单的 read() 方式采集（但效率低） |
| **Streaming** | 支持 mmap/userptr 流式采集 | **我们用这个**，效率最高 |

**怎么得出结论的：**
- 看到 `Streaming` → 代码中可以用 mmap 方式（VIDIOC_REQBUFS + mmap + VIDIOC_QBUF/DQBUF）
- 看到 `Read/Write` → 也可以用简单的 read() 方式，但我们不用（太慢）
- 如果只有 `Read/Write` 没有 `Streaming` → 就只能用 read() 方式

### 2.3 当前格式（Format Video Capture）

```
Format Video Capture:
        Width/Height      : 320/240
        Pixel Format      : 'JPEG' (JFIF JPEG)
        Field             : None
        Bytes per Line    : 320
        Size Image        : 76800
        Colorspace        : JPEG
```

**你需要关注的：**
- 这是**当前**设置的格式，不是全部支持的格式
- 默认是 320x240 JPEG，我们需要在代码中改成 640x480 YUYV
- `Field: None` → 逐行扫描（不是隔行扫描），这是我们想要的

### 2.4 V4L2 Controls（用户控制参数）

```
User Controls

                       contrast 0x00980901 (int)    : min=0 max=255 step=1 default=0 value=0 flags=slider
                     saturation 0x00980902 (int)    : min=0 max=255 step=1 default=64 value=64 flags=slider
                            hue 0x00980903 (int)    : min=0 max=359 step=1 default=0 value=0 flags=slider
        white_balance_automatic 0x0098090c (bool)   : default=1 value=1 flags=update
                    red_balance 0x0098090e (int)    : min=0 max=4095 step=1 default=0 value=0 flags=inactive, slider
                   blue_balance 0x0098090f (int)    : min=0 max=4095 step=1 default=0 value=0 flags=inactive, slider
                       exposure 0x00980911 (int)    : min=0 max=65535 step=1 default=0 value=885 flags=inactive, volatile
                 gain_automatic 0x00980912 (bool)   : default=1 value=1 flags=update
                           gain 0x00980913 (int)    : min=0 max=1023 step=1 default=0 value=156 flags=inactive, volatile
                horizontal_flip 0x00980914 (bool)   : default=0 value=0
                  vertical_flip 0x00980915 (bool)   : default=0 value=0
           power_line_frequency 0x00980918 (menu)   : min=0 max=3 default=1 value=1
```

**怎么读每一行：**

以 `contrast` 为例：
```
contrast 0x00980901 (int) : min=0 max=255 step=1 default=0 value=0 flags=slider
│        │          │       │     │       │       │         │       │
│        │          │       │     │       │       │         │       └─ flags: slider 表示适合用滑块控制
│        │          │       │     │       │       │         └─ value: 当前值是 0
│        │          │       │     │       │       └─ default: 默认值是 0
│        │          │       │     │       └─ step: 步进为 1（每次调整的最小单位）
│        │          │       │     └─ max: 最大值 255
│        │          │       └─ min: 最小值 0
│        │          └─ 类型: int（整数）
│        └─ V4L2 Control ID: 0x00980901（代码中用 V4L2_CID_CONTRAST）
└─ 名称: contrast
```

**flags 的含义（重要！）：**

| flag | 含义 | 对代码的影响 |
|------|------|-------------|
| `slider` | 适合用滑块 UI | Qt 界面中用 QSlider |
| `inactive` | 当前不可写 | **需要先关闭自动模式才能手动设置** |
| `volatile` | 值会自动变化 | 读取时每次都是最新值 |
| `update` | 修改后会影响其他 control | 比如关闭自动白平衡后，red/blue_balance 变为可写 |

**关键发现：**
- `exposure` 的 flags 是 `inactive, volatile` → 因为 `auto_exposure` 开着，曝光由硬件自动控制
- 要手动控制曝光，必须先：`setAutoExposure(false)` → 然后 `setExposure(value)`
- 同理，`gain` 也是 inactive，需要先关闭 `gain_automatic`

**对内窥镜的意义：**
- contrast/saturation → 界面上放滑块，让用户调节
- exposure → 内窥镜在体腔内光照变化大，可能需要手动控制曝光
- hflip/vflip → 内窥镜镜头方向不固定，需要翻转
- white_balance → 内窥镜 LED 光源色温偏差，可能需要手动白平衡

## 3. 解读 `--list-formats-ext` 输出

```
[0]: 'JPEG' (JFIF JPEG, compressed)
        Size: Discrete 640x480
                Interval: Discrete 0.067s (15.000 fps)
                Interval: Discrete 0.033s (30.000 fps)
                Interval: Discrete 0.017s (60.000 fps)
```

**怎么读：**

```
[0]: 'JPEG'                          ← 格式索引 0，四字符码 JPEG
     (JFIF JPEG, compressed)         ← 人类可读的描述
        Size: Discrete 640x480       ← 支持的分辨率（Discrete = 固定值，不是连续范围）
                Interval: Discrete 0.033s (30.000 fps)  ← 这个分辨率下支持的帧率
```

**怎么得出"支持 JPEG 格式"的结论：**
- 输出中有 `[0]: 'JPEG'` 这一整段 → 说明支持 JPEG
- 输出中有 `[2]: 'YUYV'` 这一整段 → 说明支持 YUYV
- 每种格式下面列出了所有支持的分辨率和帧率组合

**怎么选择参数：**

```
我们的选择过程：

1. 格式选择：YUYV
   - JPEG: 数据量小，但需要 libjpeg 解码，增加 CPU 开销和代码复杂度
   - UYVY: 和 YUYV 类似但字节序不同，OpenCV 的 cvtColor 对 YUYV 支持更好
   - YUYV: 无需解码，直接转 RGB，OpenCV 有现成的 cvtColor 函数
   - RGBP: RGB565 色深只有 16bit，色彩精度不够

2. 分辨率选择：640x480
   - 屏幕是 800x480，640x480 最接近且不超过屏幕尺寸
   - 比 1280x720 小，CPU 处理压力小
   - 是最常见的 VGA 分辨率，兼容性好

3. 帧率选择：30fps
   - 640x480 支持 15/30/60 fps
   - 30fps 足够流畅，留出 CPU 余量给图像处理
   - 60fps 虽然更流畅，但图像处理可能跟不上
```

## 4. 信息提取清单

拿到一个新摄像头时，按这个顺序提取信息：

```
第一步：确认设备存在
  ls /dev/video*

第二步：查看驱动和能力
  v4l2-ctl --device=/dev/video0 --all
  关注：Driver name, Capabilities (Streaming?), Controls

第三步：查看支持的格式
  v4l2-ctl --device=/dev/video0 --list-formats-ext
  关注：有哪些像素格式，每种格式支持哪些分辨率和帧率

第四步：确定采集参数
  格式：优先 YUYV（通用性好，OpenCV 支持好）
  分辨率：匹配显示屏幕，不要超过 CPU 处理能力
  帧率：30fps 通常是最佳平衡点

第五步：抓帧验证
  v4l2-ctl --set-fmt-video=width=640,height=480,pixelformat=YUYV \
           --stream-mmap=4 --stream-count=1 --stream-to=test.raw
  检查文件大小是否正确（width * height * 每像素字节数）
```
