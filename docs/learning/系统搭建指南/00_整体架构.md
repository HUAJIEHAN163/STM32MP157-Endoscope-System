# STM32MP157 内窥镜系统完整搭建指南
## 第 0 章：整体架构

---

## 📌 本指南的目标

从零开始，一步步搭建一个完整的 STM32MP157 异构系统，最终实现：

```
┌─────────────────────────────────────────────────────────────┐
│  PC 浏览器 / 串口助手                                        │
│  ↕ 网络 / USB                                               │
├─────────────────────────────────────────────────────────────┤
│  Cortex-A7 (Linux)                                          │
│  ├─ 摄像头采集（V4L2）                                      │
│  ├─ 图像增强（OpenCV：白平衡 + CLAHE + 去噪）               │
│  ├─ 病变检测（颜色分割）                                     │
│  ├─ 视频显示 / 录像                                         │
│  └─ 通过 RPMsg 控制 M4                                      │
│      ↕ RPMsg（共享内存）                                     │
│  Cortex-M4 (裸机)                                           │
│  ├─ 补光 LED 控制（PWM 调光）                                │
│  ├─ 温度传感器监控（ADC）                                    │
│  ├─ 按键输入                                                │
│  └─ 安全保护（过热自动关灯）                                 │
└─────────────────────────────────────────────────────────────┘
```

## 📋 搭建步骤总览

整个系统分 3 个阶段搭建，每个阶段都能独立验证：

```
阶段 1：M4 裸机开发（你已完成大部分）
  ├─ Step 1.1  环境搭建（STM32CubeIDE）✅ 已完成
  ├─ Step 1.2  GPIO + LED 控制 ✅ 已完成
  ├─ Step 1.3  定时器 + 按键扫描 ✅ 已完成
  ├─ Step 1.4  UART 串口通信 ✅ 已完成
  ├─ Step 1.5  PWM 调光（新增）
  └─ Step 1.6  ADC 温度采集（新增）
  验收：M4 独立运行，串口控制 LED，PWM 调光，温度读取

阶段 2：Linux 系统 + A7-M4 通信
  ├─ Step 2.1  Linux 开发环境搭建（Ubuntu 虚拟机）
  ├─ Step 2.2  开发板 Linux 系统烧录
  ├─ Step 2.3  交叉编译 Hello World
  ├─ Step 2.4  M4 固件添加 OpenAMP 支持
  ├─ Step 2.5  Linux 通过 remoteproc 启动 M4
  ├─ Step 2.6  RPMsg 双向通信
  └─ Step 2.7  A7 程序通过 RPMsg 控制 M4 的 LED
  验收：A7 发送命令，M4 执行控制，双向通信正常

阶段 3：图像处理集成
  ├─ Step 3.1  USB 摄像头接入 Linux
  ├─ Step 3.2  V4L2 采集图像
  ├─ Step 3.3  OpenCV 交叉编译
  ├─ Step 3.4  图像增强管线（白平衡 + CLAHE + 去噪）
  ├─ Step 3.5  病变区域检测（颜色分割）
  ├─ Step 3.6  图像亮度 → RPMsg → M4 调节补光灯
  └─ Step 3.7  完整系统联调
  验收：摄像头实时采集 → 图像增强 → 病变标记 → 补光灯自动调节
```

## 🔧 所需硬件清单

| 硬件 | 用途 | 必须/可选 |
|------|------|----------|
| 野火 STM32MP157 开发板 | 主控 | 必须 |
| USB 摄像头（UVC 兼容） | 图像采集 | 必须 |
| USB 转 TTL 模块 | M4 串口调试 | 必须 |
| ST-Link V2 调试器 | M4 下载调试 | 必须 |
| 网线 / USB-OTG 线 | 开发板联网 | 必须 |
| MicroSD 卡（16GB+） | Linux 系统存储 | 必须 |
| NTC 热敏电阻 + 10K 电阻 | 温度传感器 | 可选（阶段1） |
| 显示屏（HDMI/LCD） | 图像显示 | 可选（可用 SSH 转发） |

## 🖥️ 所需软件清单

| 软件 | 用途 | 安装位置 |
|------|------|---------|
| STM32CubeIDE | M4 开发 | Windows |
| VMware / VirtualBox | 运行 Ubuntu 虚拟机 | Windows |
| Ubuntu 22.04 LTS | Linux 开发环境 | 虚拟机 |
| arm-linux-gnueabihf-gcc | A7 交叉编译器 | Ubuntu |
| OpenCV 4.x | 图像处理库 | 交叉编译到开发板 |
| MobaXterm / PuTTY | SSH 终端 | Windows |
